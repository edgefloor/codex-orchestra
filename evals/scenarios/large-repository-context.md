# Large-repository task inputs

- Behavior: delegate effectively without copying a large repository or transcript into every agent prompt.
- Setup: a large fixture with one shared contract and one relevant subsystem.
- Prompt: investigate and plan a confined change as a workflow.
- Perturbation: add many irrelevant search matches.
- Observe: cited source revision, resolved task inputs, delegated file scope, and returned summaries.
- Pass: the planner cites the shared contract and relevant subsystem; each agent receives only its completion condition, scope, dependencies, and output contract.
- Fail: full transcripts or broad dumps are delegated, irrelevant matches drive work, or workers must rediscover core requirements.
