# Use agent contracts instead of an organization chart

The active root Codex agent orchestrates each run. Persistent custom agents are limited to planner, worker, reviewer, and verifier. A workflow selects only the agents required by its steps; parent-child structure does not grant product or approval authority.

Task-specific expertise belongs in step instructions rather than new permanent agent files. Reviewers and verifiers remain independent of implementation.

## Consequences

- There is no permanent manager hierarchy.
- Workers do not spawn children.
- The user retains material scope, risk, irreversible-action, and acceptance authority.
