# Use agent contracts instead of an organization chart

The Rust runtime orchestrates each run. Agent behavior is declared per step with explicit instructions, model, reasoning, context, outputs, and authority. Orchestra does not select persistent role personalities; parent-child structure does not grant product or approval authority.

Task-specific expertise belongs in step instructions. Assurance steps remain independent of implementation.

## Consequences

- There is no permanent manager hierarchy.
- Agent steps do not spawn children by default.
- The user retains material scope, risk, irreversible-action, and acceptance authority.
