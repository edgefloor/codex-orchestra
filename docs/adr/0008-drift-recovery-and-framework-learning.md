# Diagnose failure before retry and recover from durable evidence

Orchestra retries unchanged work only for diagnosed transient failure. Stale context is repackaged; semantic failure requires a revised assignment; late results remain evidence but cannot advance reassigned work. A Blocker requires concrete missing authority, dependency, safety, or irreversibility evidence plus an owner, impact, attempts, recovery path, and default—complexity, uncertainty, or a test failure alone is not a Blocker.

Recovery reconciles the latest accepted charter, plan, decisions, source revision, worktrees, results, and verification before choosing the next safe action. Framework learning is promoted only when a lesson is general, testable, versioned, and placed in the narrowest durable source; incident chronology does not accumulate in skills or global instructions.

## Consequences

- Drift reopens only the earliest affected Gate and dependent work.
- Handoffs contain accepted state, evidence, residual risk, and exact next action without relying on transcript continuity.
- Behavioral regressions become evaluation scenarios before framework rules are changed.
