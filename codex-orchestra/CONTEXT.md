# Codex Orchestra

Codex Orchestra is a Codex-native framework for conducting bounded software-delivery engagements through skills, custom agents, native collaboration, and repository-grounded evidence.

## Language

**Operator**:
The human who supplies intent and retains authority over material scope, risk, budget, irreversible actions, and acceptance.
_Avoid_: User, customer, administrator

**Grounding**:
A revisioned understanding of the objective, scope, acceptance, constraints, assumptions, and risks that authorizes delivery planning.
_Avoid_: Discovery phase, requirements dump

**Engagement**:
One Operator-authorized body of work conducted by Orchestra from Grounding through acceptance or handoff.
_Avoid_: Job, workflow execution

**Workstream**:
A bounded delivery slice with a named owner, dependencies, write domain, evidence contract, and integration order.
_Avoid_: Department, permanent team

**Wave**:
One dependency-ready set of assignments released together under explicit concurrency, attempt, and review capacity limits.
_Avoid_: Batch, sprint

**Context Capsule**:
The minimal revision-pinned assignment context containing the goal, done condition, binding references, write domain, validation, authority, escalation path, and output contract.
_Avoid_: Full transcript, prompt dump

**Join Owner**:
The agent accountable for waiting on a delegated branch, reviewing its evidence, integrating or rejecting its result, and reporting a bounded synthesis upward.
_Avoid_: Coordinator, observer

**Reviewer**:
An agent that independently judges an implementation against acceptance and risk and reports findings without patching the reviewed source.
_Avoid_: Verifier, repair worker

**Verifier**:
An agent that independently executes prescribed gates, records reproducible evidence, and reports residual risk without changing product source.
_Avoid_: Reviewer, implementer

**Gate**:
A deterministic, model-judgment, or human-approval condition that must be satisfied before an Engagement advances.
_Avoid_: Suggestion, checklist item

**Drift**:
A conflict between active work and a binding charter, plan, decision, interface, source revision, or observed repository fact.
_Avoid_: Any unexpected result

**Handoff**:
A transcript-independent continuation artifact containing accepted state, evidence, remaining risk, and the exact next action.
_Avoid_: Conversation summary

**Attempt**:
One bounded execution try for an assignment under a specific Context Capsule and source revision.
_Avoid_: Agent identity, task
