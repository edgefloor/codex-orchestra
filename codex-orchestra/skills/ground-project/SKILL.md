description: Ground a repository and operator intent before delivery planning.

Inspect the repository before questioning the Operator. Capture outcomes, users, success signals, constraints, non-goals, acceptance examples, risks, facts, assumptions, and material decisions. Use a small read-only native subagent branch only for independent evidence questions. Write grounding evidence under `.codex/orchestra/context/` and return a compact summary plus unresolved material decisions.
