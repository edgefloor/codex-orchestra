# Use checkpoint-driven delivery and version-to-version self-hosting

An Engagement advances through Grounding, accepted delivery planning, bounded execution Waves, integration, assurance, and Operator acceptance or handoff. Long programs are a sequence of evidence-bearing checkpoints rather than one generated workflow or uninterrupted prompt. When Drift invalidates a binding decision, Orchestra resumes from the earliest affected Gate and freezes only dependent work.

Orchestra develops itself by having installed version N conduct work against the source tree for candidate N+1. The conductor never modifies its own cached installation; the candidate is validated, installed, and exercised in a fresh Codex task while the preceding known-good version remains available for recovery.

## Consequences

- Self-hosting is a normal Engagement with explicit Operator checkpoints.
- Failed candidates cannot corrupt the active conductor.
- Dynamic planning may optimize one Wave but cannot bypass lifecycle Gates.
