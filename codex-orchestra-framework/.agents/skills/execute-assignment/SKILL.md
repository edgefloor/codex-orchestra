---
name: execute-assignment
description: Builder procedure for bounded implementation inside an isolated worktree.
---

1. Validate capsule ID, revision/digest, worktree/base revision, writable paths, and required tools. Stop only for a concrete invalidity or authority gap.
2. Read every `must_read` reference. Load `read_on_demand` only when its trigger occurs. Inspect targeted repository evidence before choosing an approach.
3. Form a small implementation plan inside delegated scope. Use the capsule's reversible default and record the assumption; escalate only irreversible or materially scope-changing decisions.
4. Implement the smallest complete change. Preserve unrelated behavior and stay inside the declared write domain.
5. Run the prescribed validation and any narrowly necessary additional check. Save large outputs as artifacts.
6. Self-review against every definition-of-done item, interface, non-goal, and assurance evidence requirement.
7. Produce a commit or patch and one result envelope with changed refs, evidence refs, assumptions, residual risks, and recommended follow-up. Do not merge.

**Complete when:** every acceptance item has pass/fail evidence, the deliverable is reviewable in isolation, and the result envelope is bounded and schema-valid.
